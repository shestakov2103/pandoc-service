# Pandoc Converter Microservice

A lightweight, isolated microservice built with **Node.js (Express)** and **Pandoc**.

Its primary purpose is to offload CPU-intensive document conversion tasks from the main application.
It accepts Markdown content via a REST API and returns a formatted DOCX file using a custom reference template and Lua filters.

## рџљЂ Features

* **Markdown to DOCX conversion** using Pandoc.
* **Custom Styling:** Uses `reference.docx` for Word styles (fonts, margins, headers).
* **Lua Filters:** Supports custom logic via `filters.lua`.
* **Dockerized:** Isolated environment with all necessary dependencies pre-installed.
* **Hot Reload:** configured for local development using `tsx` and `nodemon`.

## рџђі Docker Instructions

### 1\. Build the Image

Run this command in the root directory of the project:

```bash
docker build -t pandoc-service:latest .
```

### 2\. Run the Container

Map port **3010** (host) to **3000** (container internal):

```bash
docker run --rm -p 3010:3000 --name pandoc-service pandoc-service:latest
```

The service will be available at: `http://localhost:3010`

### 3\. Run with Custom Templates (Hot-Swap)

If you want to change styles (`reference.docx`) or filters without rebuilding the image, you can mount a local directory to `/app/templates`.

**Note:** Ensure your local folder contains `reference.docx` and `filters.lua`.

```bash
docker run --rm \
  -p 3010:3000 \
  -v $(pwd)/templates:/app/templates \
  -e PORT=3000 \
  -e MAX_REQUEST_SIZE=100mb \
  pandoc-service:latest
```

### 4\. Run in Docker Swarm

```bash
docker service create \
  --name pandoc-service \
  --replicas 1 \
  --publish published=3010,target=3000 \
  --mount type=bind,source=$(pwd)/templates,target=/app/templates \
  --env PORT=3000 \
  --env MAX_REQUEST_SIZE=100mb \
  pandoc-service:latest
```

## рџ”Њ API Documentation

### POST `/convert`

Converts Markdown text to a DOCX binary buffer.

**URL:** `http://localhost:3010/convert`
**Headers:** `Content-Type: application/json`

#### Request Body

```json
{
  "markdown": "# Title\n\nThis is a **bold** text and an image placeholder."
}
```

> **Note:** The `markdown` field supports standard Markdown syntax. Images should be pre-processed (e.g., converted to Base64) before sending if they are not external URLs.

#### Response

* **Status:** `200 OK`
* **Content-Type:** `application/vnd.openxmlformats-officedocument.wordprocessingml.document`
* **Body:** Binary file (`.docx`)

### Example Usage (cURL)

```bash
curl -X POST http://localhost:3010/convert \
  -H "Content-Type: application/json" \
  -d '{"markdown": "# Hello World\n\nGenerated by Pandoc Service."}' \
  --output result.docx
```
